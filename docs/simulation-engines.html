📊 Overview & Architecture

These 15 engines form VoronoiMap's Simulation Layer — turning static spatial tessellations into dynamic scientific models. Each engine treats Voronoi cells as autonomous spatial units (territories, demes, neurons, weather stations, tectonic plates) and simulates complex dynamics across them.

All engines share a common architecture: they accept point sets (or load from files), compute Voronoi tessellations internally, run multi-step simulations, generate autonomous insights with natural-language explanations, compute health scores (0-100), and export results as interactive HTML dashboards, JSON, or CSV.

🌿 Ecosystem Simulator

vormap_ecosystem — 636 lines

Simulates an ecosystem where each Voronoi cell is a territory occupied by competing species. Species interact via Lotka-Volterra dynamics (competition, predator-prey, mutualism) and migrate across cell boundaries.

Autonomous Features

  • Extinction Early Warning — alerts when a species drops below critical population across all cells
  • Invasive Species Alert — detects when one species dominates >70% of cells
  • Biodiversity Index — Shannon diversity per tick with decline alerts
  • Auto-Intervention (--autopilot) — corrective measures to maintain biodiversity (predator boosts, habitat restoration)
  • Ecosystem Health Score — composite 0-100 from diversity, stability, evenness

Usage

from vormap_ecosystem import EcosystemSimulator, export_html

sim = EcosystemSimulator(n_points=30, n_species=4, ticks=100)
result = sim.run()
export_html(result, "ecosystem.html")

CLI

python vormap_ecosystem.py --points 30 --species 4 --ticks 100 --output eco.html
python vormap_ecosystem.py --preset rainforest --autopilot --output result.html

🧬 Population Genetics

vormap_genetics — 740 lines

Models evolutionary dynamics across Voronoi cells where each cell acts as a local population (deme) with its own gene pool. Simulates selection, migration (gene flow), genetic drift, and mutation over generations.

Seven Analysis Engines

EnginePurpose
Gene Pool InitializerAssigns allele frequencies per cell using spatial clines plus random noise
Selection PressureApplies fitness differentials based on spatial environment zones (tropical/temperate/polar)
Migration EngineGene flow between adjacent cells inversely proportional to geographic distance
Genetic DriftStochastic sampling via binomial draws in finite populations scaled by cell area
Mutation EngineForward/reverse mutation at configurable rate, tracking mutation-selection balance
Hardy-Weinberg TesterChi-squared test for HW equilibrium deviation per cell per locus
Insight GeneratorDetects clines, isolation by distance, fixation events, speciation signals (FST), health score 0-100

Usage

from vormap_genetics import GeneticsEngine, genetics_analyze

result = genetics_analyze("points.txt")
print(f"Health: {result.health_score:.1f}/100")

engine = GeneticsEngine(points=[(0,0),(10,0),(5,8),(3,6),(7,2)])
result = engine.analyze(generations=100)
engine.to_html("genetics.html")

🔄 Metabolism Engine

vormap_metabolism — 794 lines

Models abstract resource production, consumption, and trade across Voronoi cells. Enables spatial economic analysis: which cells are producers vs consumers, where bottlenecks constrain flow, and how efficiently the system moves resources from surplus to deficit areas.

Seven Analysis Engines

  • Production Estimator — capacity from cell area × connectivity
  • Consumption Modeler — demand by proximity to spatial centroid
  • Trade Flow Computer — surplus routing to deficit neighbors (inverse distance weighting)
  • Bottleneck Detector — high pass-through, low self-sufficiency, critical bridging cells
  • Efficiency Analyzer — per-cell and system-wide resource utilization ratios
  • Metabolic Rate Calculator — throughput per unit area
  • Insight Generator — Gini coefficient of surplus, actionable recommendations

Usage

from vormap_metabolism import MetabolismEngine, metabolism_analyze

result = metabolism_analyze("points.txt")
print(f"Health: {result.health_score:.1f}/100")

⚖️ Equilibrium Analysis

vormap_equilibrium — 954 lines

Treats Voronoi cells as a physical system where each cell exerts spatial pressure on neighbors based on area imbalance, compactness difference, and centroid distance. Finds equilibrium states, classifies stability, maps basins of attraction, and detects tipping points.

Six Analysis Engines

  • Force Field Computation — net spatial force vectors per cell based on area-pressure from neighbors
  • Equilibrium Classification — identifies low-force cells; stable/unstable/saddle via local Jacobian eigenvalue analysis
  • Basin of Attraction Mapping — assigns each cell to an equilibrium via gradient-following trajectories
  • Perturbation Response Predictor — forecasts displacement propagation depth, affected cells, energy change, time-to-restabilize
  • Tipping Point Detection — finds cells near basin boundaries where small perturbations cause regime shifts
  • Interactive HTML Dashboard — 4-tab self-contained visualization

Usage

from vormap_equilibrium import analyze_equilibrium

points = [(100, 200), (400, 300), (700, 500), (200, 700), (600, 100)]
bounds = (0, 1000, 0, 1000)

report = analyze_equilibrium(points, bounds=bounds)
print(report.summary())
report.to_json("equilibrium.json")
report.to_html("equilibrium.html")

🦠 Contagion Simulator

vormap_contagion — 531 lines

Models SIR (Susceptible → Infected → Recovered) epidemic and information spread across Voronoi cells. Infection propagates through spatial adjacency with configurable transmission and recovery rates.

Autonomous Features

  • Outbreak Detection — alerts when global infection rate crosses 25%
  • Hotspot Tracker — identifies cells with highest infection each tick
  • R₀ Estimator — estimates effective reproduction number from simulation data
  • Auto-Quarantine (--autopilot) — isolates cells exceeding 50% infection by cutting migration 80%
  • Vaccination Campaign (--autopilot) — proactive S→R conversion in highest-risk neighbors after outbreak
  • Epidemic Health Score — composite 0-100 from recovery, containment, R₀

Presets

flu0.30.130Fast classic spread
zombie0.60.0220Hard to recover
rumor0.40.340Fast but loses interest
pandemic0.250.0550Slow burn

sim = ContagionSimulator(n_points=30, ticks=100, beta=0.3, gamma=0.1)

🏔️ Erosion Simulation

vormap_erosion — 394 lines

Simulates hydraulic and thermal erosion on Voronoi tessellations where each cell has an elevation. Produces time-series of per-cell elevations exportable as JSON, CSV, or animated SVG.

Two Erosion Models

ModelMechanismReal-World Analog
HydraulicMaterial removed proportional to slope to neighbors, carried downhillRiver valley formation, rainfall erosion
ThermalMaterial collapses when slope exceeds talus angle thresholdRockfall, scree slope formation

Usage

from vormap_erosion import hydraulic_erosion, thermal_erosion, export_erosion_svg

# Hydraulic erosion over 50 steps
frames = hydraulic_erosion(adjacency, elevations, steps=50, rate=0.05)

🧪 Evolutionary Optimizer

vormap_evolve — 427 lines

Uses a genetic algorithm to autonomously evolve point configurations toward user-specified spatial objectives. The optimizer acts as an intelligent agent: given a goal, it breeds, mutates, and selects configurations over generations.

Optimization Objectives

ObjectiveFitness Target
uniformMinimize coefficient of variation of cell areas
clusteredMaximize nearest-neighbor distance variance (tight groups)
coverageMaximize convex hull area relative to bounding box
spreadMaximize minimum nearest-neighbor distance
balancedMinimize max-minus-min cell area (equitable partitioning)

Usage

from vormap_evolve import evolve

best = evolve(n_points=30, width=500, height=500,
              objective="uniform", generations=200)

👁️ Attention Engine

vormap_attention — 819 lines

Decides where is interesting in a Voronoi diagram without being told. Allocates analytical focus across cells based on information density, change velocity, strategic importance, and surprise — making the diagram self-aware about which areas deserve deeper analysis.

Seven Attention Engines

  • Information Density — area variance vs neighbors, neighbor count deviation, local clustering coefficient
  • Change Velocity — displacement tracking and area flux across snapshots
  • Strategic Importance — cells at topological chokepoints: bridges between clusters, high-betweenness nodes
  • Surprise Detector — cells violating expected spatial autocorrelation (local Moran-like statistic)
  • Convergence Zones — areas where multiple attention signals overlap
  • Attention Decay — temporal inhibition-of-return: recently-attended areas suppressed, forcing exploration
  • Autonomous Scheduler — weighted priority queue producing an ordered attention schedule

Usage

from vormap_attention import AttentionEngine, attention_analyze

result = attention_analyze("points.txt")
print(f"Health: {result.health_score:.1f}/100")

engine = AttentionEngine(points=[(0,0),(10,0),(5,8)])
result = engine.analyze()
engine.to_html("attention.html")
