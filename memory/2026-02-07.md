### AgentBox Architecture Evolution

2. **v2**: Process-per-user OpenClaw instances (failed - singleton gateway conflict)
3. **v3**: Session-per-user via OpenClaw API (current)

### Key Decision: Session Isolation over Process Isolation
- Can't run multiple OpenClaw gateways (singleton scheduled task)
- Instead: single OpenClaw instance, separate sessions per user
- Each user gets unique sessionKey: `agentbox:user:<telegramId>`
- Memory isolation via per-user SQLite databases
- Trade-off: less isolation but practical on single machine
