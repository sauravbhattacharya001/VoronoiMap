## Sauravcode Compiler (evening session)

### Background
- **Sauravcode**: Shubho's custom programming language — minimal syntax, no parens/commas/semicolons
- Repo: https://github.com/sauravbhattacharya001/sauravcode
- Google Sites page: https://sites.google.com/view/sauravcode
- File extension: `.srv`
- Previously had Python interpreter (`saurav.py`): tokenizer → parser → AST → tree-walk interpreter

### Compiler Built: `sauravcc.py`
- Compiles sauravcode (.srv) → C source → gcc → native executable
- Architecture: Tokenizer → Parser → AST → CCodeGenerator → .c file → gcc
- Reused/enhanced tokenizer and parser from interpreter
- Added new AST nodes: ProgramNode, PrintNode, IfNode, WhileNode, ForNode, CompareNode, StringNode
- Fixed token ordering bug: `==` must come before `=` in regex (longest match first)
- Added comparison operators: `==`, `!=`, `<`, `>`, `<=`, `>=`
- Added `print` keyword support

### CLI Usage
```
python sauravcc.py program.srv          # Compile & run
python sauravcc.py program.srv --emit-c # Show generated C
python sauravcc.py program.srv -o app   # Custom output name
python sauravcc.py program.srv --keep-c # Keep .c file
python sauravcc.py program.srv -v       # Verbose
```

### Test Results
- `a.srv` (original example): `add`, `sub`, `fun`, `fun2` → outputs `20`, `15.87` ✅
- `test.srv`: `add`, `multiply`, `square`, `hypotenuse` → all correct ✅
- Committed and pushed to GitHub

### Known Limitation
- `factorial n - 1` parses as `factorial(n) - 1` not `factorial(n-1)` — inherent ambiguity in no-parens syntax
- Possible solutions: optional parens, or special grouping syntax
